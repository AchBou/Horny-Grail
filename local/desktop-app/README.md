# HornyGrail Desktop Uploader

Tauri desktop app for scanning a local folder, checking whether images already exist in backend metadata, and uploading originals plus thumbnails through server-issued presigned URLs.

Originals and thumbnails are content-hash based. The uploader computes the SHA-256 hash, checks backend metadata by `id`, skips duplicate originals, and can repair or regenerate thumbnails without reuploading the original file.

## Current Behavior

- Choose a local folder and watch it for new supported media files.
- Compute SHA-256 natively through the Tauri `compute_sha256_streaming` command.
- Check backend metadata and integrity before uploading.
- Upload missing originals and thumbnails through backend-issued presigned URLs.
- Repair partial assets when metadata exists but the original or thumbnail is missing.
- Regenerate thumbnails manually for already-known files.
- Generate image thumbnails natively and WebM thumbnails through a native ffmpeg path.

## Configuration

Copy `.env.example` to `.env` and provide:

- `VITE_API_BASE_URL`
- `VITE_WRITE_API_KEY`

Current `.env` shape:

```env
VITE_API_BASE_URL=https://your-api-id.execute-api.your-region.amazonaws.com/api
VITE_WRITE_API_KEY=replace-with-write-api-key
```

The desktop app no longer needs direct AWS credentials for S3 or DynamoDB access.

Upload request shape, supported MIME types, size limits, and object key rules are documented in `../../docs/upload-contract.md`. Keep desktop and mobile clients aligned with that contract before changing backend upload validation.

## Thumbnail Generation

Image thumbnails are generated by the native Tauri `generate_thumbnail` command.

WebM thumbnails are generated by the native Tauri `generate_video_thumbnail` command, which runs `ffmpeg.exe` from `src-tauri/binaries/ffmpeg.exe` on Windows. The binary is fetched on demand by `npm run ffmpeg:ensure`, and `npm run tauri -- dev` plus `npm run tauri -- build` run that step automatically through `src-tauri/tauri.conf.json`. On non-Windows, the ensure script skips the download and the command falls back to `ffmpeg` from `PATH`.

The ffmpeg path is intentionally native because browser video/canvas thumbnail generation can hang in Tauri WebView media events. The browser-based generator remains as a fallback after native generation fails.

When updating ffmpeg:

- Update the pinned download settings in `scripts/ensure-ffmpeg.mjs`.
- Delete `src-tauri/binaries/ffmpeg.exe` locally and rerun `npm run ffmpeg:ensure`.
- Run `npm run tauri -- build` to verify the binary is copied into the release resources and installers.
- Review ffmpeg license obligations before distributing the desktop app.
